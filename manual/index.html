1. 앱 개요

1.1 앱 소개

알림온나는 식품/생활용품/소모품 등의 구매일과 만료일을 관리하고, 알림·쇼핑·구매내역·달력·AI 스마트 등록을 한 앱에서 처리하는 생활 관리 도구입니다.

이 앱은 별도의 외부 운영 서버 없이 사용자 기기 안의 로컬 데이터를 기반으로 동작합니다. 개인 정보가 외부로 유출되지 않는다는 장점이 있지만, 반대로 다른 사람과 같은 데이터를 실시간으로 함께 공유하는 용도로는 적합하지 않습니다.

1.2 지원 환경

현재는 안드로이드 앱으로 출시되어 있으며, iOS 버전은 준비 중입니다. 테블릿은 정식 지원 대상은 아니지만, 테블릿에서도 실행할 수 있도록 최소한의 화면 대응은 적용되어 있어 기본적인 사용은 가능합니다.

1.3 주요 기능

만료일 알림 카테고리/태그 관리 쇼핑 리스트 구매 내역 달력 보기 AI 영수증 분석

1.4 화면 이동

메인 네비게이션에서 전체/알림 대상/기한 만료/소비 완료 필터를 빠르게 오갈 수 있으며, 카테고리·태그 기반으로 세부 분류가 가능합니다.

1.5 구매일과 만료일

구매일과 만료일은 앱 설계의 핵심 기준이지만, 실제 사용 목적에 맞게 다르게 해석해 사용할 수 있습니다. 예를 들어 구매일은 등록일처럼 사용할 수 있고, 만료일은 소비기한처럼 사용할 수 있습니다. 이 용어(라벨)는 설정에서 사용자가 원하는 표현으로 직접 바꿀 수 있습니다.

2. 빠른 시작

  1. 메인 화면 우측 상단 +로 첫 품목을 등록합니다.
  2. 설정 > 알림에서 알림 서비스 ON, 시간, 기본 알림일(예: 3일 전)을 지정합니다. (기본 알림일은 신규 항목의 기본값이며, 실제 알림일은 항목별로 개별 변경 가능)
  3. 필요하면 설정 > 확장 기능 > AI API 키 등록에서 키를 저장하고 연결 테스트를 수행합니다.
  4. 설정 > 데이터 관리에서 Google Drive 백업을 1회 만들어 둡니다.
첫 운영 루틴 권장: 등록 → 알림 확인 → 소비/정리 → 쇼핑 리스트 반영

3. 메인 화면 사용법

3.1 네비게이션

  • 기본 필터: 전체 보기, 알림 대상, 기한 만료, 소비 완료
  • 카테고리와 활성 태그를 좌측 네비게이션에서 바로 선택
  • AI 키가 1개 이상 등록되어 있으면 스마트 등록(Beta) 메뉴 노출
  • 하위 화면 바로 이동: 쇼핑 리스트, 구매 내역, 달력, 설정

3.2 목록 조작

  • 검색: 품목명 + 메모(노트) 대상 검색
  • 정렬: 입력순, 이름순, 구매일순, 만료일순 + 오름/내림차순
  • 다중 선택 모드: 선택 품목 일괄 쇼핑 리스트 추가 / 카테고리 이동 / 삭제

3.3 가독성 제스처

  • 목록 영역에서 핀치로 글자 배율 조절(약 90~130%)
  • 테블릿/넓은 화면은 분할 내비게이션, 모바일은 오버레이 내비게이션 사용

4. 품목 등록/편집

4.1 등록 항목

  • 기본: 품목명, 카테고리, 구매일, 만료일
  • 추가: 수량/단위, 가격, 태그, 메모, 이미지, 알림 사용 여부/알림일(항목별 개별 설정 가능)
  • 기한 없음: 만료일의 연도가 9999년으로 저장된 항목은 앱 내부에서 기한 없음으로 처리합니다.

4.2 편집 화면 기능

  • 저장, 초기화(신규/복제 모드), 복제 등록, 쇼핑 리스트 보내기, 삭제
  • 연속 편집 모드가 켜져 있으면 여러 품목을 같은 화면에서 페이지 이동하며 편집할 수 있으며, 입력란 하단의 여백 부분을 좌우로 스와이프해 이전/다음 항목으로 이동할 수 있습니다.
  • 빠른 입력(Quick Input) 다이얼로그로 텍스트 한 줄에서 이름/수량/가격/날짜/메모를 추출 (가격은 숫자만 입력 가능)
빠른 입력 예시: 우유 2개 4500 3.3 3.10 냉장칸
날짜는 3.3 같은 짧은 형식도 입력 가능합니다. (예: 3.3, 03.03, 2026.3.3)

4.3 수량/단위 입력 상세

  • 개념(재고 관리): 수량은 보유량 숫자, 단위는 기준 단위(예: 개/봉/팩/병)입니다.
  • 입력법: 2개, 1.5L, 12처럼 입력할 수 있습니다. 숫자와 단위는 붙여 써도 되고 띄어 써도 인식됩니다.
  • 단위 변경: 기존 값이 2개일 때 2병처럼 숫자+새 단위를 직접 입력하면 단위가 바뀝니다.
  • 자동 단위 유지: 단위를 한 번 입력한 뒤 숫자만 수정하면 마지막 단위를 자동 유지해 재입력을 줄여줍니다.
  • -/+ 버튼: 수량 형식이 인식될 때만 활성화됩니다. -는 최소 0까지 감소, +는 증가하며, 정수는 1단위/소수는 0.1단위로 증감합니다.
예: 3팩에서 숫자만 5로 바꾸면 5팩처럼 단위가 유지됩니다.

4.4 태그 입력 상세

  • 제안 팝업: 태그 입력 중 기존 태그와 일치/포함되는 항목이 있으면 제안 목록이 표시되며, 탭해서 바로 추가할 수 있습니다.
  • 신규 태그 추가: 원하는 텍스트를 입력한 뒤 스페이스(공백)를 누르면 새 태그가 즉시 추가됩니다. (예: 냉장 입력 후 공백)
  • 중복 방지: 이미 추가된 동일 태그는 중복으로 다시 추가되지 않습니다.
  • 삭제: 추가된 태그 칩의 x 버튼으로 개별 삭제할 수 있습니다.
  • 일괄 선택: 입력칸 오른쪽 ... 버튼으로 태그 선택 창을 열어 여러 태그를 한 번에 선택/해제할 수 있습니다.

5. 달력 화면

  • 보기 모드 전환: 만료일 기준 / 구매일 기준
  • 월 단위 탐색 + 오늘로 즉시 이동
  • 날짜별 품목 수, 상태 색상(임박/만료 등) 표시
  • 특정 날짜 선택 시 하단 리스트 또는 상세 화면에서 품목 확인/편집 이동

달력 시작 요일, 폰트 범위, 달력 높이 비율/아이템 스케일은 설정에서 조정할 수 있습니다.

참고: 권장 폰트 크기 범위는 폰 기준 날짜 10~14, 항목 8~10, 테블릿 기준 날짜 12~16, 항목 10~14입니다.
달력 화면에서 항목 폰트 크기는 추가적으로 조절이 가능합니다.

6. 쇼핑 리스트

6.1 핵심 기능

  • 그룹 단위 리스트 운영(추가/이름 변경/정렬/삭제)
  • 개별/전체 선택 체크, 구매 여부 토글
  • 선택 항목 일괄 작업: 삭제, 그룹 이동, 구매일/구매처 변경, 구매내역 저장
  • 디스미서블 제스처: 항목을 좌우로 밀어 삭제 또는 관리대상에 등록 작업을 빠르게 실행

6.2 빠른 입력 및 연동

  • 쇼핑 항목 등록 시 항목명에 타이핑하면 구매내역을 조회해 추천 리스트를 표시하고, 선택 시 이름/수량/가격/구매처를 빠르게 채울 수 있습니다.
  • 쇼핑리스트 등록/편집 화면의 가격 입력란 * 버튼은 수량과 가격을 곱해 계산한 값을 가격칸에 반영합니다.
  • 빠른 입력: 한줄로 이어 입력하면 이름/수량/가격을 추출해서 추가합니다.

6.3 칩 모드

  • 쇼핑리스트는 리스트 모드와 칩 모드를 오갈 수 있으며, 하단 그룹 칩 영역을 위/아래로 드래그해 전환할 수 있습니다.
  • 칩 모드에서는 항목이 카드형 칩으로 표시되고 화면이 구매대상 / 구매완료 두 영역으로 나뉩니다.
  • 칩을 다른 영역으로 드래그하면 구매 상태가 바로 변경되어, 체크박스 대신 끌어다 놓는 방식으로 정리할 수 있습니다.
  • 칩을 아래쪽의 다른 그룹 칩으로 드래그하면 해당 그룹으로 옮길 수 있으며, 기본 동작은 이동입니다.
  • 드래그한 상태에서 다른 손가락으로 화면을 함께 터치한 채 그룹에 놓으면 복사로 처리되어, 원래 그룹 항목은 유지하고 대상 그룹에 동일 항목을 추가할 수 있습니다.
  • 가운데 구분선은 위아래로 드래그해 두 영역의 높이 비율을 조절할 수 있습니다.
  • 칩을 오른쪽 아래 삭제 버튼으로 드래그하면 해당 쇼핑 항목을 빠르게 삭제할 수 있습니다.
메인 화면에서 여러 품목을 선택한 후 쇼핑 리스트로 바로 전송하면 재고 보충 루틴을 빠르게 만들 수 있습니다.

7. 구매 내역

  • 검색(품목명/구매처), 날짜 그룹 헤더 표시, 무한 스크롤 로딩
  • 수동 추가/수정 가능
  • 전체 선택/해제, 선택 삭제, 특정 날짜 이전 일괄 삭제 지원

쇼핑 리스트나 스마트 등록에서 항목을 구매 내역으로 저장하면 히스토리 누적이 자동화됩니다.

8. 스마트 등록(Beta)

8.1 동작 흐름

  1. 그룹 선택 또는 생성
  2. 카메라/갤러리로 영수증 이미지 선택 후 크롭
  3. 영수증 이미지에서 문자를 추출 하고, AI 분석 요청
  4. 인식 항목 검토/수정 후 메인 품목으로 변환 등록

8.2 화면 보조 기능

  • 이미지-리스트 분할 높이 드래그 조절
  • 인식 박스 하이라이트, 항목 포커스 확대
  • 일괄 구매일 변경, 그룹 초기화, 구매 내역 저장

8.3 동시 처리와 백그라운드 대기

  • 스마트 등록 분석은 작업그룹별로 독립적으로 돌아가므로, 서로 다른 그룹의 영수증 분석을 동시에 진행할 수 있습니다.
  • 한 그룹의 AI 응답이 올 때까지 스마트 등록 화면에서 계속 기다릴 필요는 없으며, 다른 화면으로 이동해 일반 목록/쇼핑리스트/구매내역 등 다른 작업을 이어서 할 수 있습니다.
  • 분석이 진행 중이면 네비게이션 메뉴의 스마트 등록(Beta) 항목 앞에 스피너가 표시되어, 현재 백그라운드에서 처리 중인 작업이 있음을 알려줍니다.
  • 분석이 끝나면 해당 그룹으로 다시 들어왔을 때 결과/오류 안내를 확인할 수 있습니다.
인식 후 리스트에서 항목명 아래 붉은 물결선이 보이면, 분류/구매일/소비기한 같은 필수 입력 항목이 아직 비어 있다는 뜻입니다.
필요한 값을 모두 입력하면 붉은 줄이 사라지고, 관리대상에 등록할 수 있는 상태가 됩니다.
영수증 이미지는 외부 AI 서비스로 직접 전송되지 않으며, 먼저 기기 내부에서 텍스트를 추출한 뒤 필요한 텍스트 정보만 분석에 사용합니다.
앱을 완전히 닫거나, 폰이 앱을 강제로 새로고침/중지하는 경우에는 진행 중이던 백그라운드 분석이 중단될 수 있습니다.
스마트 등록 메뉴는 AI API 키가 있을 때만 메인 네비게이션에 표시됩니다.

9. 설정 가이드

설정 화면은 카테고리별 분리 + 검색 기능을 제공하며, 현재 주요 항목은 아래와 같습니다.

화면 및 테마 설정 테마 선택, 커스텀 테마 편집, 앱 글꼴, 언어, 줄 간격, 앱 글자 크기, 구매일/소비기한 라벨명 변경, 달력 폰트 범위, 품목 이미지 표시, 네비게이션 헤더 이미지 사용/선택/삭제, 네비게이션 아이콘 표시, 쇼핑리스트 그룹 칩 한 줄 표시
알림 관리 알림 서비스 ON/OFF, 시스템 알림 권한 상태 확인 및 설정 화면 이동, 알림 시간, 기본 알림일(신규 항목 기본값), 만료 항목 포함 여부
날짜 및 동작 설정 날짜 형식, 날짜 구분자, 달력 시작 요일, 자동 저장 대상(메인 품목/쇼핑리스트/구매내역), 연속 편집 모드
분류 및 이미지 관리 카테고리 관리, 태그 관리, 쇼핑리스트 그룹 관리, 이미지 보관함, 이미지 해상도, 이미지 품질
스마트 기능 및 위젯 AI API 키 관리(OpenAI/Gemini/Claude), 기본 모델/엔드포인트 등 AI 설정, 연결 테스트, 홈 위젯 터치 동작 설정
데이터 관리 Google Drive 백업/복원, 엑셀 내보내기, 엑셀 가져오기
앱 정보 및 시스템 앱 정보, 개인정보처리방침, 오픈소스 라이선스, 저장소 사용량 확인

10. Google Drive 백업/복원

10.1 백업

  • Google 계정 연결 후 백업 파일명 지정
  • 설정 포함 여부 선택 가능
  • 설정 포함 시 복원 정책(병합/교체) 선택

10.2 복원

  • Drive 백업 목록에서 파일 선택 후 복원
  • 복원 시 내부적으로 안전 백업 생성 후 적용
  • 복원 완료 후 앱을 재시작하지 않고 바로 사용할 수 있도록 상태를 갱신하지만, 안정성을 위해 앱 재실행을 권장

11. 엑셀 가져오기/내보내기

11.1 내보내기

  • 설정 > 데이터 관리 > 엑셀 내보내기 실행 시 앱 데이터가 .xlsx로 생성되어 공유됩니다.
  • 기본 열: 카테고리, 품목명, 수량, 가격, 구매일, 만료일, 메모, 등록일
중요: 엑셀 내보내기 파일은 백업 파일이 아닙니다.
이 파일만으로 앱 데이터를 원래 상태로 복원할 수 없습니다.
전체 복원이 필요하면 Google Drive 백업/복원 기능을 사용하세요.

11.2 가져오기

  • .xlsx만 지원, 첫 번째 시트만 읽음
  • 필수 열: 품목명(Item Name), 구매일(Purchase Date), 만료일(Expiry Date)
  • 선택 열: 카테고리, 수량, 가격, 메모
  • 임시 테이블 미리보기 후 실제 저장 단계에서 반영
가장 안전한 방식은 앱의 내보내기 파일 형식을 그대로 유지해 다시 가져오는 것입니다.

12. 데이터/보안

  • 기본 데이터는 로컬 DB(SQLite) + 앱 문서 폴더 파일로 관리
  • 앱 문서 폴더에는 DB, 이미지, 테마 파일이 포함됩니다.
  • AI API 키는 기기의 보안 저장 영역에 암호화되어 저장됩니다.
API 키는 보안 특성상 일반 백업/복원 흐름에 포함되지 않으므로, 기기 교체 시 재입력이 필요할 수 있습니다.

13. 문제 해결

Q1. 알림이 오지 않습니다.

  • 앱 설정의 알림 서비스가 켜져 있는지 확인
  • 시스템 알림 권한이 허용되어 있는지 확인
  • 알림 시간, 기본 알림일(디폴트값), 항목별 개별 알림일을 함께 확인

Q2. 스마트 등록 메뉴가 보이지 않습니다.

  • AI API 키가 최소 1개 등록되어 있는지 확인
  • 확장 기능의 AI API 키 화면에서 연결 테스트 성공 여부 확인

Q3. 복원 후 데이터가 기대와 다릅니다.

  • 복원 시 설정 포함 여부/병합·교체 정책이 의도와 맞았는지 확인
  • 필요 시 최근 백업으로 다시 복원하거나 안전 백업 상태를 점검

Q4. 엑셀 가져오기가 중간에 실패합니다.

  • 첫 시트 사용 여부, 필수 열 존재 여부 확인
  • 날짜 형식(YYYYMMDD, YYMMDD, YYYY-MM-DD 등) 유효성 확인
  • 한 행에 오류가 있으면 해당 행에서 중단될 수 있으므로 데이터 정제 후 재시도

Q5. 날짜 형식과 금액 형식을 바꿀 수 있나요?

  • 기본적으로 날짜와 금액은 시스템 언어와 앱에서 설정한 언어의 로케일 기준에 따라 처리됩니다.
  • 날짜 형식은 설정에서 직접 바꿀 수 있으며, 날짜를 해석할 때도 앱에 저장된 날짜 형식 설정이 우선 적용됩니다.

Q6. 기능이 제대로 실행되지 않거나 오류가 발생합니다.

  • 여러 기기와 환경에서 테스트를 충분히 하지 못해 일부 기기에서는 예상하지 못한 문제가 생길 수 있습니다.
  • 문제가 반복되면 사용한 기능, 기기 정보, 오류 상황을 정리해 개발자 연락처로 알려주시면 앱 기능 개선에 많은 도움이 됩니다.
UI 문구/정책은 버전 업데이트에 따라 달라질 수 있습니다.
